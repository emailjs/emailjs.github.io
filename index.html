email.js

A toolbox for writing email applications in JavaScript.

What is it?

Email.js is a set of components that can be used to write email apps in pure JavaScript on the client in HTML5 or on the server in node.js. They can be used together or separately. Components include but are not limited to:

  1. an IMAP client: browserbox
  2. an SMTP client: smtpclient
  3. an email parser: mimeparser
  4. an email builder: mailbuild

Features

  • Open Source (MIT license)
  • Well tested
  • Standards based (use Typed Arrays, Raw Sockets, and the TextEncoder api)

Dependencies

In order to communicate over TCP the IMAP and SMTP components require apis that are currently only supported natively by Firefox OS. For platforms like Chrome Apps, node.js and node-webkit that use non-standard apis, the following shims are used:

  • tcp-socket
  • stringencoding

Getting started

For a list of all components and usage examples check out the Whiteout.io GitHub repos. Components that belong to email.js are MIT licensed.

Authors and Contributors

The components were originally written by Andris Reinman (@andris9), the author of Nodemailer. They are maintained by him and Felix Hammerl (@felixhammerl) for production use in Whiteout Mail. Mozilla is also planning to use them in their Firefox OS mail client.
